Copilot SDK を体験するExperience the Copilot SDK

アプリと同じエージェントランタイムを使い、内製ツール、リリースノート生成、サポート業務への組み込みを独自スタックなしで試します。Use the same agent runtime that powers the app to try in-house tools, release-note generation, and support-workflow embedding without assembling your own stack.

この機能でできることWhat you'll experience

体験のゴールYour goal

このガイドを終えると、Copilot SDK の対応言語を把握し、最小スクリプトから小さな自動化と既存ワークフローへの組み込みまでの流れを説明できるようになります。By the end of this guide you will know the supported Copilot SDK languages and be able to describe the path from a minimal script to a small automation embedded in an existing workflow.

同じランタイムを使うUse the same runtime

Copilot App を動かすエージェントランタイムを自分のツールから利用する考え方を掴む。Understand how to use the agent runtime that powers the Copilot App from your own tools.

小さな自動化を作るBuild a small automation

リリースノート生成のような繰り返し作業を SDK で組み立てる。Assemble repeatable work such as release-note generation with the SDK.

多数のサーフェスへ展開Reach many surfaces

同じ実装を内製ツール、CLI、サポート業務など複数の入口に広げる。Expand the same implementation to in-house tools, CLI surfaces, support workflows, and more.

ハンズオン手順Step-by-step

SDK で最小の自動化を作るBuild a minimal automation with the SDK

上から順に進めてください。各ステップは数分で完了します。Follow the steps in order — each takes just a few minutes.

  1. 言語Language

    使う言語を 1 つ選ぶChoose one language

    SDK は Node.js / TypeScript、Python、Go、.NET、Rust、Java で一般提供（GA）されています。手元で依存関係を管理しやすい言語を 1 つ選びます。The SDK is generally available (GA) for Node.js / TypeScript, Python, Go, .NET, Rust, and Java. Choose one language whose dependencies are easy for you to manage locally.

  2. 導入Install

    選んだ言語の SDK をインストールするInstall the SDK for your language

    公式ドキュメントに従い、選んだ言語のパッケージマネージャーで Copilot SDK を追加します。パッケージ名や API 名はバージョンで変わる可能性があるため、手元のドキュメントを正とします。Following the official documentation, add the Copilot SDK with the package manager for your language. Treat your local documentation as authoritative because package and API names can change by version.

  3. 最小Minimal

    エージェントランタイムを呼び出す最小スクリプトを書くWrite a minimal script that invokes the agent runtime

    リポジトリ名、対象ブランチ、実行したい依頼文を入力にして、エージェントランタイムへ処理を渡す最小スクリプトを作ります。具体的なメソッド名は SDK のドキュメントに合わせます。Create a minimal script that accepts a repository, target branch, and instruction, then passes the work to the agent runtime. Match concrete method names to the SDK documentation.

  4. 自動化Automate

    リリースノート生成を小さく試すTry small release-note generation

    直近の変更や PR タイトルを入力にして、リリースノートの下書きを生成する自動化を作ります。まずは出力をファイルではなく画面に表示し、内容を確認します。Use recent changes or PR titles as input and build an automation that drafts release notes. Start by printing the output to the screen instead of writing files, then inspect it.

  5. 組み込みEmbed

    既存ワークフローに組み込む場所を決めるDecide where to embed it in an existing workflow

    サポートチケット、社内ダッシュボード、CLI、チャット運用など、既存の入口から同じエージェント処理を呼び出す場所を 1 つ選びます。Choose one existing surface such as support tickets, an internal dashboard, a CLI, or chat operations where the same agent behavior can be invoked.

  6. サーフェスSurfaces

    「一つのランタイム、多数のサーフェス」を確認するConfirm “one runtime, many surfaces”

    最小スクリプトと組み込み先を比べ、同じランタイムを複数の体験に展開できることを確認します。必要ならガイド 10 の CLI やガイド 14 の拡張性へ進みます。Compare the minimal script with the embedding target and confirm that the same runtime can power multiple experiences. Continue to guide 10 for CLI or guide 14 for extensibility if useful.

確認ポイントWhat you should see

うまくいったサインSigns it worked

チェックリストChecklist

  • 対応言語として Node.js / TypeScript、Python、Go、.NET、Rust、Java を正しく把握できた。You correctly identified Node.js / TypeScript, Python, Go, .NET, Rust, and Java as supported languages.
  • 選んだ言語で SDK 依存関係を追加し、最小スクリプトの構成を作れた。You added the SDK dependency for your chosen language and created the shape of a minimal script.
  • リリースノート生成など、具体的な小さな自動化の入出力を確認できた。You inspected inputs and outputs for a concrete small automation such as release-note generation.
  • 同じエージェントランタイムを複数の入口へ展開する設計を説明できる。You can explain how the same agent runtime can be exposed through multiple surfaces.
ヒント・応用Tips & variations

もう一歩使いこなすGo a little further

試してみるThings to try

  • Copilot CLI（ガイド 10）と組み合わせると、SDK で作った処理をターミナル利用者向けの入口にしやすくなります。Combine this with Copilot CLI (guide 10) to make SDK-built behavior accessible to terminal users.
  • MCP、Skills、Plugins などの拡張性（ガイド 14）を使う場合も、同じランタイムを中心に責務を整理します。When using extensibility such as MCP, Skills, or Plugins (guide 14), organize responsibilities around the same runtime.
  • 最初から大きな社内エージェントを作らず、入力、出力、権限境界が明確な 1 つの業務から始めます。Do not start with a large internal agent; begin with one workflow whose inputs, outputs, and permission boundaries are clear.
SDK の価値は、アプリと同じ基盤を社内の文脈へ持ち込めることです。独自スタックを増やす前に、既存の運用へ小さく接続します。The SDK's value is bringing the same foundation as the app into your internal context. Before adding a custom stack, connect it to an existing workflow in a small way.
