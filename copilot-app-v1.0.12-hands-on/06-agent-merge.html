Agent Merge を体験するExperience Agent Merge

プルリクエストをレビュー、CI、必須レビュアー、マージ条件の充足まで運び、どこまで任せるかを利用者が選べます。Carry a pull request through review, CI, required reviewers, and merge conditions while choosing how much to delegate.

この機能でできることWhat you'll experience

体験のゴールYour goal

このガイドを終えると、PR を Agent Merge に渡し、CI やレビューコメントを監視・対応させ、条件が整ったら自動マージへ進める流れを理解できます。By the end of this guide you'll understand how to hand a PR to Agent Merge, let it watch and address CI or review comments, and proceed to auto-merge when conditions are met.

CI をグリーンへ戻すGet CI back to green

失敗したチェックを見つけ、必要な修正と再確認をエージェントに任せます。Let the agent find failing checks, make needed fixes, and verify again.

レビュー対応を進めるRespond to review

レビューコメントを追い、回答やコード変更の候補を作らせます。Track review comments and have candidate replies or code changes prepared.

マージ条件を追跡Track merge conditions

必須レビュアー、チェック、保護ルールを見ながら、満たされたらマージへ進めます。Watch required reviewers, checks, and protections, then move to merge once satisfied.

ハンズオン手順Step-by-step

Agent Merge を実際に使ってみるPut Agent Merge to work

上から順に進めてください。各ステップは数分で完了します。Follow the steps in order — each takes just a few minutes.

  1. 対象Select

    対象の PR を Agent Merge に渡すHand a PR to Agent Merge

    マージしたいプルリクエストを開き、Agent Merge に引き継ぎます。目的、許可する変更範囲、手動確認が必要な条件を短く伝えます。Open the pull request you want merged and hand it to Agent Merge. Briefly state the goal, allowed change scope, and conditions that require manual approval.

  2. 委任Delegate

    任せる範囲を選ぶChoose the delegation level

    CI の監視だけ、レビューコメントへの回答まで、条件を満たした後の自動マージまでなど、任せる深さを選びます。必要なら各段階で承認を挟みます。Choose how far to delegate: CI monitoring only, responding to review comments, or auto-merge after conditions are met. Add approval gates where needed.

  3. CICI

    失敗チェックを調べてグリーンに戻すInvestigate failing checks and return CI to green

    Agent Merge に CI の失敗を監視させ、ログやテスト結果から原因を絞らせます。修正案を確認し、必要な変更を行わせて再実行結果を待ちます。Let Agent Merge watch CI failures and narrow down causes from logs or test results. Review the proposed fix, let it make needed changes, and wait for the rerun.

  4. レビューReview

    レビューコメントに対応するRespond to review comments

    未対応のレビューコメントを整理し、回答だけで済むものとコード変更が必要なものに分けます。ガイド 08 の Copilot コードレビュー結果も同じ PR 文脈で扱えます。Triage unresolved review comments into items that need only a reply and items requiring code changes. Results from guide 08 Copilot code review can be handled in the same PR context.

  5. 条件Conditions

    必須レビュアーとマージ条件を追跡するTrack required reviewers and merge conditions

    必須レビュアーの承認、必須チェック、ブランチ保護などを確認し、残っている条件を Agent Merge に追跡させます。Check required reviewer approvals, required checks, branch protection, and other conditions, then have Agent Merge track what remains.

  6. マージMerge

    条件が満たされたら自動マージするAuto-merge once conditions are met

    委任レベルで許可している場合、すべての条件が満たされた時点で自動マージへ進めます。マージ後は PR の状態と関連 Issue の更新を確認します。If your delegation level allows it, proceed to auto-merge once all conditions are met. After merge, confirm PR state and related issue updates.

チェックリストChecklist

  • Agent Merge が対象 PR の CI、レビュー、マージ条件を継続的に追跡している。Agent Merge continuously tracks CI, reviews, and merge conditions for the target PR.
  • 失敗したチェックへの対応方針や修正結果が確認でき、CI がグリーンへ戻る。You can see the response plan and fix result for failing checks, and CI returns to green.
  • レビューコメントへの回答またはコード変更が整理され、未対応項目が減る。Review-comment replies or code changes are organized, reducing unresolved items.
  • 必須条件を満たした後、許可した委任範囲に従ってマージへ進める。After required conditions are met, the PR can proceed to merge according to your allowed delegation scope.
ヒント・応用Tips & variations

もう一歩使いこなすGo a little further

試してみるThings to try

  • 最初は自動マージまで任せず、CI 修復とレビューコメント整理までに限定すると挙動を把握しやすいです。At first, stop short of auto-merge and limit delegation to CI repair and review-comment triage so behavior is easier to understand.
  • レビューコメントが多い PR は、ガイド 08 の Copilot コードレビューで論点を先に整理してから Agent Merge に渡すと効率的です。For PRs with many comments, use guide 08 Copilot code review to clarify issues before handing the PR to Agent Merge.
  • 定期的にマージ待ち PR を巡回したい場合は、ガイド 05 の Automations でリマインドや要約を作ります。If you want regular sweeps of PRs waiting to merge, use guide 05 Automations for reminders or summaries.
Agent Merge の価値は「マージボタンを押す」ことだけではなく、PR が止まりがちな CI・レビュー・必須条件を継続的に前へ進めることです。Agent Merge is not only about pressing the merge button; its value is continuously moving CI, review, and required conditions forward when PRs stall.
