クラウドセッションを体験するExperience cloud sessions

GitHub がホストする分離環境でエージェントに作業を継続させ、どのデバイスからでも同じ状態を引き継いでリモート操作します。Let agents keep working in a GitHub-hosted isolated environment, then pick up the same state from any device and drive it remotely.

この機能でできることWhat you'll experience

体験のゴールYour goal

このガイドを終えると、セッションをクラウド環境で走らせ、ローカル端末に縛られずに状態を引き継いで操作できることを確認できます。By the end of this guide you can run a session in the cloud and verify that you can continue driving it without being tied to your local machine.

GitHub ホスト環境で実行Run in GitHub hosting

エージェントが分離されたクラウド環境で作業を継続する流れを確認する。See how an agent continues work in an isolated cloud environment.

別デバイスから再開Resume from another device

同じセッション状態を、別の端末やブラウザーから引き継ぐ。Pick up the same session state from another device or browser.

ローカル制約から解放Free local resources

重い処理や長時間の作業をローカルマシンに依存させない。Avoid depending on your local machine for heavier or longer-running work.

前提Prerequisites

始める前にBefore you begin

用意するものWhat you need

  • GitHub Copilot App v1.0.12 をインストール済み（Windows / macOS / Linux）GitHub Copilot App v1.0.12 installed (Windows / macOS / Linux)
  • 有効な有料 Copilot サブスクリプション（Pro / Pro+ / Max / Business / Enterprise）でサインイン済みSigned in with an active paid Copilot subscription (Pro / Pro+ / Max / Business / Enterprise)
  • 少なくとも 1 つの GitHub リポジトリへのアクセス（Issue や PR がいくつかあると体験しやすい）Access to at least one GitHub repository (a few issues or PRs make it easier to experience)
  • ローカルでコードを動かす場合は Git がインストール済みGit installed if you plan to run code locally
  • クラウドセッションを作成できるリポジトリ権限と、安定したネットワーク接続Repository permissions that allow cloud sessions and a stable network connection
ハンズオン手順Step-by-step

クラウドでセッションを継続するContinue a session in the cloud

上から順に進めてください。各ステップは数分で完了します。Follow the steps in order — each takes just a few minutes.

  1. 開始Start

    セッションを選んでクラウドで開始するChoose a session and start it in the cloud

    My Work から対象リポジトリのタスクを開き、新しいセッションを GitHub ホストのクラウド環境として開始します。既存セッションをクラウドに移せる場合は、その操作を選びます。From My Work, open a task in the target repository and start a new session in a GitHub-hosted cloud environment. If an existing session can be moved to cloud, choose that action.

  2. 確認Confirm

    分離された環境で動いていることを確認するConfirm it is running in an isolated environment

    セッションの表示や実行ログを見て、作業がローカル端末ではなく GitHub ホスト環境で進んでいることを確認します。Check the session view or run log and confirm the work is progressing in GitHub hosting rather than on your local machine.

  3. 継続Continue

    ローカル端末を離れても作業を続けるLet it keep working while you step away

    長めの調査、テスト、修正依頼をエージェントに任せ、アプリを閉じたり端末をスリープさせたりしてもセッションが継続する前提で進めます。Give the agent a longer investigation, test, or fix request, then proceed knowing the session can continue even if you close the app or put the device to sleep.

  4. 再開Resume

    別デバイスから同じセッションを開くOpen the same session from another device

    同じ GitHub アカウントで別のデバイスから Copilot App を開き、My Work でクラウドセッションを見つけて再開します。状態・ログ・次の入力先が引き継がれていることを確認します。Open Copilot App on another device with the same GitHub account, find the cloud session in My Work, and resume it. Confirm the state, log, and next input point are preserved.

  5. 操作Drive

    リモートから指示を追加するDrive it remotely

    別デバイスから追加の指示や確認を送り、エージェントが同じクラウド環境で作業を続けることを確認します。Cloud agent の使いどころはガイド 11 でも深掘りします。Send additional instructions or checks from the other device and confirm the agent continues in the same cloud environment. Guide 11 goes deeper on when to use cloud agents.

  6. 資源Resources

    ローカルリソースに依存しないことを観察するObserve independence from local resources

    重いビルドや複数ステップの検証中でも、ローカル CPU、メモリ、ディスク容量に直接依存しない体験になることを確認します。During heavier builds or multi-step verification, observe that the experience is not directly constrained by your local CPU, memory, or disk space.

確認ポイントWhat you should see

うまくいったサインSigns it worked

チェックリストChecklist

  • My Work にクラウドで実行中のセッションが表示される。My Work shows a session running in the cloud.
  • アプリを閉じたり別デバイスに移ったりしても、セッションの状態と履歴を再開できる。After closing the app or moving to another device, you can resume the session state and history.
  • 追加指示が同じクラウド環境に届き、エージェントが継続して応答する。Additional instructions reach the same cloud environment and the agent continues responding.
  • 長めの処理中でも、ローカル端末のリソース消費に左右されにくい。Longer work is less affected by local device resource usage.
ヒント・応用Tips & variations

もう一歩使いこなすGo a little further

試してみるThings to try

  • クラウドセッションは My Work に集約して見えるため、まずガイド 01 の一覧から状態を確認すると迷いません。Cloud sessions are visible in My Work, so start from the guide 01 overview when checking status.
  • ローカルで並列実行するガイド 02 と比べ、長時間・高負荷・別デバイス引き継ぎが必要な作業をクラウドに寄せると効果的です。Compared with local parallel work in guide 02, cloud is effective for long-running, heavier, or cross-device handoff tasks.
  • クラウドエージェントの権限、環境、運用の考え方はガイド 11（クラウドエージェント）でさらに確認します。Guide 11 (Cloud agent) expands on permissions, environments, and operating practices for cloud agents.
