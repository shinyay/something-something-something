Copilot コードレビューを体験するExperience Copilot code review

エージェント的なレビューでノイズを減らし、medium tier、/security-review、/rubberduck を使って重要な指摘に集中します。Use agentic review to filter noise, then focus on important findings with medium tier, /security-review, and /rubberduck.

この機能でできることWhat you'll experience

体験のゴールYour goal

このガイドを終えると、PR に Copilot レビューを依頼し、高推論モデル・セキュリティ特化レビュー・複数モデル批評を使い分けられるようになります。By the end of this guide you will be able to request Copilot review on a PR and choose among higher-reasoning, security-focused, and multi-model critique passes.

ノイズを減らすレビューNoise-filtered review

重要度の低い指摘に埋もれず、修正判断が必要な箇所に集中する。Avoid getting buried in low-value comments and focus on findings that need a decision.

高推論モデルの活用Higher-reasoning review

medium tier で精度と再現性を高めたレビューを試す。Try medium tier review for better accuracy and reproducibility.

専門パスの追加Specialized passes

/security-review と /rubberduck で観点を切り替える。Switch review perspectives with /security-review and /rubberduck.

前提Prerequisites

始める前にBefore you begin

用意するものWhat you need

  • GitHub Copilot App v1.0.12 をインストール済み（Windows / macOS / Linux）GitHub Copilot App v1.0.12 installed (Windows / macOS / Linux)
  • 有効な有料 Copilot サブスクリプション（Pro / Pro+ / Max / Business / Enterprise）でサインイン済みSigned in with an active paid Copilot subscription (Pro / Pro+ / Max / Business / Enterprise)
  • 少なくとも 1 つの GitHub リポジトリへのアクセス（Issue や PR がいくつかあると体験しやすい）Access to at least one GitHub repository (a few issues or PRs make it easier to experience)
  • ローカルでコードを動かす場合は Git がインストール済みGit installed if you plan to run code locally
  • レビュー対象にできるプルリクエスト、または小さな変更を含むブランチがあることA pull request to review, or a branch with a small change that can be reviewed
ハンズオン手順Step-by-step

Copilot レビューを PR で試すTry Copilot review on a PR

上から順に進めてください。各ステップは数分で完了します。Follow the steps in order — each takes just a few minutes.

  1. PRPR

    レビュー対象のプルリクエストを開くOpen the pull request to review

    Copilot App からレビューしたい PR を開きます。差分が大きすぎる場合は、小さな PR を選ぶと指摘の質を確認しやすくなります。Open the PR you want to review from the Copilot App. If the diff is very large, choose a smaller PR so it is easier to inspect review quality.

  2. 依頼Request

    Copilot にレビューを依頼するRequest a Copilot review

    PR の差分を対象に Copilot レビューを依頼します。UI 名称は環境で変わる可能性があるため、レビュー依頼またはエージェントレビューに相当する操作を選びます。Request a Copilot review for the PR diff. Because UI names can vary, choose the action that corresponds to requesting review or agentic review.

  3. 推論Reason

    medium tier を選ぶChoose medium tier

    利用できる場合は medium tier（高推論モデル）を選び、通常のレビューより精度と再現性が上がるかを比較します。When available, choose medium tier (a higher-reasoning model) and compare whether accuracy and reproducibility improve over a default review.

  4. 安全Security

    /security-review を実行するRun /security-review

    同じ PR で /security-review を実行し、認証、入力検証、権限、シークレット取り扱いなどセキュリティ観点に絞った指摘を確認します。On the same PR, run /security-review and inspect findings focused on security concerns such as auth, input validation, permissions, and secret handling.

  5. 批評Critique

    /rubberduck で実装を批評するCritique the implementation with /rubberduck

    /rubberduck（GA）を実行し、複数モデルによる実装批評を読みます。通常レビューと異なる観点が出るかを比べます。Run /rubberduck (GA) and read the multi-model critique of your implementation. Compare whether it surfaces perspectives that differ from the regular review.

  6. 展開Extend

    Azure DevOps のサポートを確認するNote Azure DevOps support

    GitHub 以外のチームで使う場合は、Azure DevOps でもネイティブ対応していることをメモし、同じレビュー方針を展開できるか確認します。For teams outside GitHub, note that native Azure DevOps support is available and consider whether the same review approach can be rolled out there.

チェックリストChecklist

  • PR の差分に対して Copilot のレビューコメントまたは要約が表示された。Copilot review comments or a summary appeared for the PR diff.
  • medium tier のレビューで、重要な指摘が再現しやすく整理されていることを確認できた。The medium tier review organized important findings in a way that was easier to reproduce.
  • /security-review がセキュリティ観点に絞った別パスとして実行できた。/security-review ran as a separate pass focused on security.
  • /rubberduck による複数モデル批評を、通常レビューと比較できた。You compared the multi-model critique from /rubberduck with the regular review.
ヒント・応用Tips & variations

もう一歩使いこなすGo a little further

試してみるThings to try

  • Agent Merge（ガイド 06）に渡す前に Copilot レビューを実行し、対応すべき論点を先に減らします。Run Copilot review before handing work to Agent Merge (guide 06) so there are fewer issues to resolve later.
  • Canvas（ガイド 04）でレビュー観点や判断メモを可視化すると、チーム内で合意しやすくなります。Use Canvas (guide 04) to visualize review criteria and decision notes so the team can align more easily.
  • セキュリティの指摘は、修正可否だけでなくリスク、影響範囲、テスト方法も合わせて確認します。For security findings, check not only whether you can fix them, but also risk, blast radius, and test strategy.
Copilot コードレビューは人間のレビューを置き換えるものではなく、重要な論点を早く見つける補助線です。最終判断はチームの基準に照らして行います。Copilot code review does not replace human review; it helps surface important issues earlier. Make final decisions against your team's standards.
