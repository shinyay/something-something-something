並列エージェント開発を体験するExperience parallel agent development

各ローカルセッションを独立した Git worktree で動かし、ブランチやファイルの衝突なしに複数エージェントを同時に走らせます。Run each local session in its own Git worktree so several agents can work at once without branch or file conflicts.

この機能でできることWhat you'll experience

体験のゴールYour goal

このガイドを終えると、My Work から複数のセッションを立ち上げ、それぞれが独立した worktree で安全に並走していることを確認できます。By the end of this guide you can launch multiple sessions from My Work and verify that each runs safely in its own independent worktree.

セッションを並走させるRun sessions side by side

別々のタスクやブランチで複数のエージェントを同時に進める。Move multiple agents forward at once on different tasks or branches.

worktree の分離を確認Confirm worktree isolation

各セッションがブランチの実コピーとして別ディレクトリで動くことを観察する。Observe each session running in a separate directory as a real copy of the branch.

衝突なしで切り替えるSwitch without conflicts

作業中のファイルやブランチをぶつけずにセッション間を移動する。Move between sessions without colliding files or branches.

ハンズオン手順Step-by-step

並列セッションを安全に走らせるRun parallel sessions safely

上から順に進めてください。各ステップは数分で完了します。Follow the steps in order — each takes just a few minutes.

  1. 起点Start

    My Work から最初の作業を開くOpen the first task from My Work

    ガイド 01 と同じように My Work を開き、対象リポジトリの Issue、PR、または新しい作業を選んでローカルセッションを開始します。Open My Work as in guide 01, choose an issue, PR, or new task in the target repository, and start a local session.

  2. 追加Add

    別タスクでもう 1 つセッションを始めるStart another session for a different task

    同じリポジトリまたは別リポジトリで、別のタスクを選んで 2 つ目のセッションを開始します。必要に応じて異なるブランチ名や目的を指定します。Choose a different task in the same or another repository and start a second session. Use a different branch name or goal when appropriate.

  3. 分離Isolate

    各セッションの worktree を観察するObserve each session's worktree

    セッション詳細やローカルファイルの場所を確認し、それぞれが独立した Git worktree として別々の作業コピーを持っていることを確認します。Check the session details or local file location and confirm that each one has its own working copy as an independent Git worktree.

  4. 切替Switch

    セッション間を行き来するMove between sessions

    片方のセッションでファイルを開いたまま、もう片方のセッションに切り替えます。ブランチや作業中ファイルが上書きされず、状態がそれぞれ保持されることを確認します。Leave files open in one session and switch to the other. Confirm that branches and in-progress files are not overwritten and each state is preserved.

  5. 後始末Clean up

    完了したセッションを閉じるClose a completed session

    作業が終わったセッションを終了またはアーカイブし、アプリが不要になった worktree を自動的に片付ける流れを確認します。Finish or archive a completed session and observe how the app cleans up worktrees that are no longer needed.

  6. 確認Verify

    ブランチ衝突がないことを確認するVerify there are no branch collisions

    リポジトリのブランチ一覧やセッション表示を見て、各作業が別ブランチ・別 worktree として扱われ、同じファイル変更が意図せず混ざっていないことを確認します。Check the repository branch list or session view and verify each task is treated as a separate branch and worktree, with no unintended mixing of file changes.

確認ポイントWhat you should see

うまくいったサインSigns it worked

チェックリストChecklist

  • 同時に 2 つ以上のセッションが表示され、それぞれ別のタスク名やブランチで進んでいる。Two or more sessions appear at the same time, each progressing under a different task name or branch.
  • 各セッションのローカルパスまたは詳細が、別々の worktree を示している。Each session's local path or details indicate a different worktree.
  • 一方のセッションで変更したファイルが、もう一方の作業コピーに勝手に現れない。Files changed in one session do not unexpectedly appear in the other working copy.
  • 完了後、不要になった worktree がアプリの管理下で片付く。After completion, unneeded worktrees are cleaned up under app management.
ヒント・応用Tips & variations

もう一歩使いこなすGo a little further

試してみるThings to try

  • My Work を発射台にすると、並走中のセッションを見失いにくくなります。→ ガイド 01 で一覧ビューを復習できます。Using My Work as the launch pad makes parallel sessions easier to track → revisit guide 01 for the overview.
  • 互いに依存しない小さな Issue から始めると、並列化の効果と安全性を確認しやすくなります。Start with small independent issues to see the benefit and safety of parallelization clearly.
  • ローカル環境の負荷が気になる作業は、ガイド 03 のクラウドセッションと組み合わせて試します。For work that may strain local resources, try combining this with cloud sessions in guide 03.
worktree は「同じブランチを見ている別タブ」ではなく、実際の作業コピーです。だからこそ複数エージェントを安心して並走できます。A worktree is a real working copy, not just another tab looking at the same branch. That is what makes parallel agents safe to run.
