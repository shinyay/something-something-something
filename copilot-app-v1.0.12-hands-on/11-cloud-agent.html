クラウドエージェントを体験するExperience the cloud agent

スケジュールや GitHub イベントで動き、Issue 起票やコメントを許可制から始めて autopilot へ育てるクラウドエージェントを体験します。Try a cloud agent that runs on schedules or GitHub events, starts with permission prompts for issues and comments, and can grow into autopilot.

この機能でできることWhat you'll experience

体験のゴールYour goal

このガイドを終えると、クラウドエージェントをスケジュールまたは GitHub イベントで動かし、既定の書き込み許可確認から autopilot への切り替えまでを理解できます。By the end of this guide you can run a cloud agent on a schedule or GitHub event and understand the path from default write permission prompts to autopilot.

スケジュールとイベントSchedules and events

定期実行または GitHub イベントへの応答としてクラウドでエージェントを動かします。Run the agent in the cloud on a schedule or in response to GitHub events.

書き込み前の許可Permission before writes

既定では Issue 起票やコメントなどの各書き込み操作の前に許可を確認します。By default, each write action such as filing an issue or commenting asks for permission first.

autopilot への移行Moving to autopilot

信頼できる流れになったら、対話型から autopilot へ切り替えて任せる範囲を広げます。Once the flow is trustworthy, switch from interactive operation to autopilot and expand what it can handle.

ハンズオン手順Step-by-step

クラウドエージェントを実際に動かすRun a cloud agent in practice

上から順に進めてください。各ステップは数分で完了します。Follow the steps in order — each takes just a few minutes.

  1. 設定Configure

    実行条件を選ぶChoose when it runs

    クラウドエージェントを作成し、定期的なスケジュールで動かすか、Issue や PR などの GitHub イベントに反応させるかを選びます。Create a cloud agent and choose whether it runs on a recurring schedule or responds to GitHub events such as issues or pull requests.

  2. 依頼Request

    Issue 起票またはコメントを依頼するAsk it to file an issue or comment

    対象リポジトリで、調査結果を Issue として起票する、または既存スレッドにコメントするよう依頼します。書き込みを含む小さなタスクから始めます。In the target repository, ask it to file findings as an issue or comment on an existing thread. Start with a small task that includes a write action.

  3. 許可Approve

    既定の許可確認で書き込みを承認するApprove writes through the default permission prompt

    既定では各書き込み操作の前に確認が入ります。内容、対象リポジトリ、投稿先を確認してから承認し、意図しない書き込みを防ぎます。By default, each write action asks for confirmation. Review the content, target repository, and destination before approving to avoid unintended writes.

  4. 信頼Trust

    対話型から autopilot へ切り替えるSwitch from interactive to autopilot

    同じ種類のタスクで期待どおりの結果が続くことを確認したら、必要に応じて autopilot モードに切り替えます。interactive は確認しながら進め、autopilot はより自律的に進めます。After the same type of task produces reliable results, switch to autopilot when appropriate. Interactive mode proceeds with checkpoints; autopilot proceeds more autonomously.

  5. 監視Monitor

    My Work から状態を追うTrack status from My Work

    実行中、待機中、完了したクラウドエージェントの作業を My Work で確認します。詳細な俯瞰はガイド 01 の My Work と組み合わせます。Use My Work to watch cloud-agent work that is running, waiting, or complete. Pair this with guide 01, My Work, for the broader overview.

チェックリストChecklist

  • スケジュールまたは GitHub イベントをきっかけにクラウドエージェントが開始される。The cloud agent starts from a schedule or GitHub event.
  • Issue 起票やコメントの前に、既定の許可確認が表示される。The default permission prompt appears before filing an issue or comment.
  • 承認後、指定したリポジトリに Issue またはコメントが作成される。After approval, an issue or comment is created in the selected repository.
  • 信頼できるタスクでは autopilot へ切り替え、My Work で進行状況を追える。For trusted tasks, you can switch to autopilot and track progress in My Work.
ヒント・応用Tips & variations

もう一歩使いこなすGo a little further

試してみるThings to try

  • 最初はコメントだけ、次に Issue 起票という順に、書き込みの影響が小さいタスクから始める。Start with lower-impact writes such as comments, then move to filing issues.
  • ガイド 05 の Automations と組み合わせると、定期的な調査やレポート作成の流れを設計しやすくなります。Combine this with guide 05, Automations, to design recurring investigation or reporting flows.
  • クラウドで動く作業の引き継ぎや確認は、ガイド 03 の Cloud sessions と一緒に試すと理解しやすいです。Try this alongside guide 03, Cloud sessions, to understand handoff and review for cloud-based work.
autopilot は「確認を省く」ためではなく、十分に信頼できる定型タスクをより自律的に進めるためのモードとして扱うと安全です。Treat autopilot not as a way to skip review, but as a mode for letting well-understood, trusted tasks proceed more autonomously.
