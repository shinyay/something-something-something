Canvas（キャンバス）を体験するExperience Canvas

人とエージェントが同じ作業面を見ながら、プラン・PR・ターミナル・ブラウザー・ワークフロー状態をその場で更新し、編集・承認・軌道修正できます。A shared work surface where people and agents see plans, PRs, terminals, browsers, and workflow state together, then update, edit, approve, and course-correct in place.

この機能でできることWhat you'll experience

体験のゴールYour goal

このガイドを終えると、セッション中に Canvas を開き、エージェントの進行を見ながらプランを編集し、次の一手をその場で承認できるようになります。By the end of this guide you'll be able to open Canvas during a session, watch an agent progress, edit its plan, and approve the next move in place.

双方向の作業面A two-way work surface

人とエージェントが同じ Canvas 上で計画、実行状況、判断ポイントを共有します。People and agents share plans, execution state, and decision points on the same Canvas.

ライブに更新される状態Live-updating state

プラン、PR、ブラウザー、ターミナル、デプロイ、ダッシュボード、ワークフロー状態の変化を追えます。Track changes across plans, PRs, browsers, terminals, deployments, dashboards, and workflow state.

その場で介入Intervene in place

項目の編集・並べ替え・承認・軌道修正を、別画面に移らず実行します。Edit, reorder, approve, and course-correct without switching away.

前提Prerequisites

始める前にBefore you begin

用意するものWhat you need

  • GitHub Copilot App v1.0.12 をインストール済み（Windows / macOS / Linux）GitHub Copilot App v1.0.12 installed (Windows / macOS / Linux)
  • 有効な有料 Copilot サブスクリプション（Pro / Pro+ / Max / Business / Enterprise）でサインイン済みSigned in with an active paid Copilot subscription (Pro / Pro+ / Max / Business / Enterprise)
  • 少なくとも 1 つの GitHub リポジトリへのアクセス（Issue や PR がいくつかあると体験しやすい）Access to at least one GitHub repository (a few issues or PRs make it easier to experience)
  • ローカルでコードを動かす場合は Git がインストール済みGit installed if you plan to run code locally
  • Canvas を表示できるセッション、またはエージェントに任せたい小さな作業があることA session that can show Canvas, or a small task you want to hand to an agent
ハンズオン手順Step-by-step

Canvas を実際に使ってみるPut Canvas to work

上から順に進めてください。各ステップは数分で完了します。Follow the steps in order — each takes just a few minutes.

  1. 開始Start

    セッションを開いて Canvas を表示するOpen a session and show Canvas

    Copilot App でリポジトリのセッションを開き、エージェントの作業状況を見られる Canvas を表示します。表示位置や呼び出し方は環境で異なるため、セッション内の作業面として開きます。Open a repository session in Copilot App and show the Canvas surface where agent work can be viewed. Placement can vary, so open it as the session's working surface.

  2. 観察Observe

    エージェントが作るプランやビューを見るWatch the plan and views the agent populates

    小さな修正や調査を依頼し、Canvas にプラン、PR の状態、ターミナル出力、ブラウザー確認などが追加・更新される様子を見ます。Ask for a small fix or investigation and watch Canvas populate or update with a plan, PR state, terminal output, browser checks, and related views.

  3. 編集Edit

    項目を編集・並べ替えるEdit and reorder items

    プランの順序が意図と違う場合は、Canvas 上で項目の文言や順序を調整します。エージェントに先に確認してほしいステップを上に移します。If the plan order does not match your intent, adjust wording or ordering directly on Canvas. Move the step you want checked first to the top.

  4. 承認Approve

    提案された次のステップを承認するApprove the proposed next step

    エージェントが実行前に確認を求めたら、内容を読み、問題なければ承認します。危険そうな操作や範囲が広い操作は、より小さい単位に分けるよう指示します。When the agent asks before acting, read the proposal and approve it if it looks right. For risky or broad changes, instruct it to split the work into smaller steps.

  5. 修正Course-correct

    その場で軌道修正するCourse-correct in place

    途中で目的がずれたら、Canvas 上の現在地を見ながら、期待する結果・除外したい範囲・確認方法を短く追記します。If the work drifts, use the current Canvas state to add a brief correction: expected outcome, scope to avoid, and how to verify.

  6. 共有Share

    ライブの共有面として使うUse it as a live shared surface

    レビューやペア作業では、Canvas を見ながら「どこまで進んだか」「何を承認待ちか」をそろえます。ガイド 08 のコードレビュー結果も同じ文脈で扱えます。During review or pairing, use Canvas to align on what is done and what is waiting for approval. Results from guide 08 code review fit the same context.

チェックリストChecklist

  • Canvas 上にプランや作業状態が表示され、エージェントの進行に合わせて更新される。Canvas shows a plan and work state that update as the agent progresses.
  • プラン項目の文言や順序を変更し、その変更が以後の作業に反映される。You can change plan wording or ordering, and the change is reflected in later work.
  • エージェントの提案ステップを承認、またはより安全な手順へ軌道修正できる。You can approve an agent proposal or steer it toward a safer sequence.
  • PR、ターミナル、ブラウザー、ワークフロー状態などを一つの作業面で参照できる。You can reference PR, terminal, browser, and workflow state from one work surface.
ヒント・応用Tips & variations

もう一歩使いこなすGo a little further

試してみるThings to try

  • 大きな依頼は、Canvas 上で「調査」「変更」「検証」「PR」のように段階へ分けるとレビューしやすくなります。For larger requests, split the Canvas plan into phases such as investigation, change, verification, and PR.
  • ターミナルやブラウザーの結果だけで判断せず、PR 差分やテスト結果も同じ Canvas 上で確認します。Do not decide from terminal or browser results alone; check PR diffs and test results on the same Canvas.
  • CLI での操作と組み合わせたい場合は、ガイド 10 の Copilot CLI と見比べると役割分担が分かります。If you want to combine it with CLI work, compare guide 10 Copilot CLI to understand the division of labor.
Canvas は AX（Agent Experience）への第一歩です。人がエージェントを監督するための「見える場所」を作ることが価値の中心です。Canvas is a first step toward AX (Agent Experience): its core value is giving people a visible place to supervise agents.
