パートナーエージェントアプリを体験するExperience partner agent apps

サードパーティのエージェントアプリを Copilot と連携し、Issue を特定のエージェントに任せて自動化・コード生成・分析・アクション実行を体験します。Connect third-party agent apps with Copilot, assign an issue to a specific agent, and experience automation, code generation, context analysis, and actions.

この機能でできることWhat you'll experience

体験のゴールYour goal

このガイドを終えると、パートナーエージェントアプリを追加し、GitHub Issue を個別のエージェントにアサインして、結果をレビューできるようになります。By the end of this guide you'll be able to add a partner agent app, assign a GitHub issue to an individual agent, and review what it produced.

パートナーを選ぶChoose a partner

LaunchDarkly、Sonar、PagerDuty、Miro、Amplitude、Endor Labs、Octopus Deploy などの連携候補を確認します。Review partner options such as LaunchDarkly, Sonar, PagerDuty, Miro, Amplitude, Endor Labs, and Octopus Deploy.

Issue を任せるDelegate an issue

GitHub Issue を特定のエージェントにアサインし、作業対象と期待する成果を明確にします。Assign a GitHub issue to a specific agent and make the target work and expected result clear.

結果をレビューするReview the result

自動化、生成されたコード、分析、実行されたアクションを人間が確認して取り込む流れを体験します。Review the automation, generated code, analysis, and actions before you adopt the result.

前提Prerequisites

始める前にBefore you begin

  • 有効な有料 Copilot サブスクリプション（Pro / Pro+ / Max / Business / Enterprise）でサインイン済みSigned in with an active paid Copilot subscription (Pro / Pro+ / Max / Business / Enterprise)

  • ローカルでコードを動かす場合は Git がインストール済みGit installed if you plan to run code locally
ハンズオン手順Step-by-step

パートナーエージェントに Issue を任せるDelegate an issue to a partner agent

上から順に進めてください。各ステップは数分で完了します。Follow the steps in order — each takes just a few minutes.

  1. 探すBrowse

    利用できるパートナーエージェントを確認するBrowse available partner agents

    Copilot App や GitHub 側の連携画面で、利用できるパートナーエージェントアプリを確認します。LaunchDarkly、Sonar、PagerDuty、Miro、Amplitude、Endor Labs、Octopus Deploy など、目的に近いものを選びます。In the Copilot App or GitHub integration flow, review available partner agent apps. Choose one that matches your goal, such as LaunchDarkly, Sonar, PagerDuty, Miro, Amplitude, Endor Labs, or Octopus Deploy.

  2. 接続Connect

    アプリを追加して承認するAdd and authorize the app

    選んだパートナーアプリを追加し、必要なリポジトリや組織へのアクセスを承認します。最小限の権限から始め、作業対象のリポジトリだけを許可します。Add the partner app you selected and authorize access to the required repository or organization. Start with the smallest useful scope and allow only the repository you need.

  3. 準備Prepare

    エージェント向けの Issue を整えるPrepare an issue for the agent

    GitHub Issue に目的、対象ファイルやサービス、制約、完了条件を書きます。エージェントが自動化、コード生成、コンテキスト分析、アクション実行のどれを行うべきかを明記します。In the GitHub issue, write the goal, target files or services, constraints, and completion criteria. State whether the agent should automate a task, generate code, analyze context, or take an action.

  4. 割り当てAssign

    Issue を特定のエージェントにアサインするAssign the issue to a specific agent

    Issue の担当先として、追加したパートナーエージェントを選びます。人間の担当者と同じように、必要ならコメントで補足のコンテキストを渡します。Assign the issue to the partner agent you added. As you would for a human assignee, add a comment with extra context if needed.

  5. 確認Review

    エージェントの作業結果を確認するReview what the agent did

    エージェントが残したコメント、提案、生成コード、外部サービスでのアクション結果を確認します。必要に応じて Copilot の通常セッションで追加修正し、最終判断は人間が行います。Review the agent's comments, suggestions, generated code, and any action results in the external service. If needed, continue in a regular Copilot session and keep the final decision human-owned.

確認ポイントWhat you should see

うまくいったサインSigns it worked

チェックリストChecklist

  • パートナーエージェントアプリが対象リポジトリまたは組織に接続されている。The partner agent app is connected to the target repository or organization.
  • Issue の担当先として、特定のパートナーエージェントを選べた。You could select a specific partner agent as the issue assignee.
  • エージェントが自動化、コード生成、分析、または外部サービス上のアクションを実行した記録が残っている。There is a record that the agent automated a task, generated code, analyzed context, or took an action in an external service.
  • 結果を人間がレビューし、採用・修正・差し戻しを判断できる状態になっている。A human can review the result and decide whether to accept, adjust, or send it back.
ヒント・応用Tips & variations

もう一歩使いこなすGo a little further

試してみるThings to try

  • 最初は小さく安全な Issue で試し、エージェントがどの範囲まで自律的に動くかを確認する。Start with a small, safe issue and observe how independently the agent acts.
  • 外部サービスの専門性が必要な Issue は、通常のコードセッションよりパートナーエージェントが向く場合があります。クラウド実行との違いはガイド 11 で確認できます。Issues that need external-service expertise may fit partner agents better than a regular code session. Compare cloud execution in guide 11.
  • 複数の連携を使う場合は、どの Issue をどのエージェントに任せたかを My Work で追跡すると見失いにくくなります。When using several integrations, track which issue went to which agent from My Work so nothing gets lost.
パートナーエージェントは「Copilot に外部サービスの専門エージェントを参加させる」入口です。権限は必要最小限にし、結果は必ずレビューしてから取り込みます。Partner agents are the entry point for bringing external-service specialists into Copilot. Keep permissions minimal and always review results before adopting them.
