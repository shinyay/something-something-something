拡張性（MCP / Skills / Plugins）を体験するExperience extensibility (MCP / Skills / Plugins)

MCP サーバー、カスタムスキル、プラグイン、gist で共有できる拡張機能を使い、自分たちの標準・社内システム・文脈をエージェントやレビューに持ち込みます。Use MCP servers, custom skills, plugins, and gist-shareable extensions to bring your standards, internal systems, and context into agents and reviews.

この機能でできることWhat you'll experience

体験のゴールYour goal

このガイドを終えると、Copilot App に外部コンテキストや独自ルールを追加し、それをエージェント作業やレビューで活用する流れを説明・試行できるようになります。By the end of this guide you'll be able to add external context and custom rules to the Copilot App and try using them in agent work or reviews.

MCP で接続するConnect with MCP

MCP サーバー接続で、社内システムや開発ツールの情報を Copilot から参照できるようにします。Use MCP server connections so Copilot can reference information from internal systems or engineering tools.

スキルとプラグインを足すAdd skills and plugins

カスタムエージェントスキルとマーケットプレイスのプラグインで、チーム固有の作業手順を補強します。Strengthen team-specific workflows with custom agent skills and marketplace plugins.

拡張機能を共有するShare extensions

Canvas 拡張を含む gist 共有可能な拡張機能で、標準化した体験をチームへ展開します。Roll out standardized experiences to a team with gist-shareable extensions, including Canvas extensions.

前提Prerequisites

始める前にBefore you begin

用意するものWhat you need

  • GitHub Copilot App v1.0.12 をインストール済み（Windows / macOS / Linux）GitHub Copilot App v1.0.12 installed (Windows / macOS / Linux)
  • 有効な有料 Copilot サブスクリプション（Pro / Pro+ / Max / Business / Enterprise）でサインイン済みSigned in with an active paid Copilot subscription (Pro / Pro+ / Max / Business / Enterprise)
  • 少なくとも 1 つの GitHub リポジトリへのアクセスと、MCP・スキル・プラグイン・拡張機能を追加できる権限Access to at least one GitHub repository and permission to add MCP connections, skills, plugins, or extensions
  • ローカルでコードを動かす場合は Git がインストール済みGit installed if you plan to run code locally
ハンズオン手順Step-by-step

独自コンテキストを Copilot に追加するAdd your own context to Copilot

上から順に進めてください。各ステップは数分で完了します。Follow the steps in order — each takes just a few minutes.

  1. MCPMCP

    MCP サーバー接続を追加するAdd an MCP server connection

    接続したい社内システムや開発ツールの MCP サーバー情報を用意し、Copilot App の拡張設定から接続します。認証が必要な場合は、表示される承認フローに従います。Prepare the MCP server details for the internal system or engineering tool you want to connect, then add it from the Copilot App extensibility settings. If authentication is required, follow the authorization flow shown.

  2. スキルSkill

    カスタムエージェントスキルを追加するAdd a custom agent skill

    チームのコーディング標準、レビュー観点、運用手順などをスキルとして追加します。最初は短い説明と明確な使用条件に絞ると、エージェントが使い分けやすくなります。Add team coding standards, review criteria, or operational procedures as a skill. Start with a short description and clear usage conditions so the agent can choose it appropriately.

  3. プラグインPlugin

    マーケットプレイスからプラグインを入れるInstall a marketplace plugin

    必要な連携がマーケットプレイスにある場合は、提供元と権限を確認してインストールします。レビューやエージェントが参照できる情報の範囲を把握しておきます。If the integration you need is available in the marketplace, review the provider and permissions before installing it. Understand what information reviews or agents can access.

  4. 拡張Extension

    gist 共有可能な拡張機能を追加するInstall a gist-shareable extension

    共有された gist から拡張機能をインストールします。Canvas 拡張の場合は、表示される UI や入力項目がチームの目的に合っているかを確認します。Install an extension from a shared gist. For a Canvas extension, check that the UI and inputs match the team's goal.

  5. 活用Use

    エージェントやレビューで文脈を使うUse the context in an agent or review

    小さな Issue やレビュー対象を開き、追加した MCP 接続、スキル、プラグイン、拡張機能を前提に作業させます。出力にチーム標準、社内情報、エンジニアリング文脈が反映されているか確認します。Open a small issue or review target and ask Copilot to work with the MCP connection, skill, plugin, or extension you added. Check whether the output reflects your team standards, internal information, and engineering context.

チェックリストChecklist

  • MCP 接続、スキル、プラグイン、または拡張機能が Copilot App に追加されている。An MCP connection, skill, plugin, or extension has been added to the Copilot App.
  • 追加した要素の権限や参照範囲を確認できている。You have checked the permissions and access scope of what you added.
  • エージェントまたはレビューの出力に、独自標準・社内システム・エンジニアリング文脈が反映されている。Agent or review output reflects your standards, internal systems, or engineering context.
  • 不要になった拡張を無効化・削除できる場所を把握している。You know where to disable or remove an extension when it is no longer needed.
ヒント・応用Tips & variations

もう一歩使いこなすGo a little further

試してみるThings to try

  • 社内情報を扱う MCP 接続は、読み取り専用から始めると検証しやすくなります。For MCP connections that touch internal information, start read-only when possible to make validation easier.
  • Canvas 拡張の体験設計はガイド 04、開発者向け API の考え方はガイド 09 と合わせて確認すると理解しやすくなります。Review guide 04 for Canvas experience design and guide 09 for developer-facing API concepts.
  • スキルは長大な規約集にせず、エージェントが判断に使う短いチェックリストとして始めるのがおすすめです。Do not start with a huge policy document; begin with a short checklist the agent can actually use for decisions.
拡張性は「Copilot を自分たちの開発環境に近づける」ための機能です。接続先と権限を明確にし、レビュー可能な小さな単位で増やすと安全に育てられます。Extensibility is how you make Copilot fit your engineering environment. Keep destinations and permissions clear, then grow capability in small, reviewable increments.
