Copilot CLI を体験するExperience the Copilot CLI

ターミナル派向けに再設計された CLI で、タブ付き TUI、ローカル音声モード、定期プロンプトとバックグラウンドタスクを体験します。Use the redesigned CLI for terminal fans to try the tabbed TUI, local voice mode, recurring prompts, and background tasks.

この機能でできることWhat you'll experience

体験のゴールYour goal

このガイドを終えると、Copilot CLI を起動し、/experimental、音声モード、/every を使ってターミナルから作業を進められます。By the end of this guide you can launch the Copilot CLI and use /experimental, voice mode, and /every to work from the terminal.

再設計された TUIRedesigned TUI

/experimental で PR・Issue・gist にタブでアクセスします。Use /experimental for tabbed access to PRs, issues, and gists.

端末内の音声入力On-device voice input

音声モードではオンデバイス音声認識により、音声が端末外へ出ません。Voice mode uses on-device speech recognition so audio stays on your machine.

定期実行とバックグラウンドRecurring and background work

/every で定期プロンプトやバックグラウンドタスクをスケジュールします。Use /every to schedule recurring prompts and background tasks.

前提Prerequisites

始める前にBefore you begin

用意するものWhat you need

  • GitHub Copilot App v1.0.12 と Copilot CLI を利用できる状態GitHub Copilot App v1.0.12 and the Copilot CLI available to use
  • 有効な有料 Copilot サブスクリプション（Pro / Pro+ / Max / Business / Enterprise）でサインイン済みSigned in with an active paid Copilot subscription (Pro / Pro+ / Max / Business / Enterprise)
  • 少なくとも 1 つの GitHub リポジトリへのアクセス（Issue や PR があると確認しやすい）Access to at least one GitHub repository (issues or PRs make it easier to verify)
  • ローカル作業用の Git とターミナル、音声モードを試す場合はマイクGit and a terminal for local work, plus a microphone if you try voice mode
ハンズオン手順Step-by-step

Copilot CLI を実際に使ってみるPut the Copilot CLI to work

上から順に進めてください。各ステップは数分で完了します。Follow the steps in order — each takes just a few minutes.

  1. 起動Launch

    CLI をインストールまたは更新して起動するInstall or update the CLI and launch it

    公式の案内に従って Copilot CLI を用意し、ターミナルから起動します。サインインを求められたら、Copilot App と同じ GitHub アカウントで認証します。Follow the official instructions to prepare the Copilot CLI, then launch it from your terminal. If prompted, authenticate with the same GitHub account you use in the Copilot App.

  2. TUITUI

    /experimental でタブ付きビューを開くOpen the tabbed view with /experimental

    CLI の会話内で /experimental を入力し、再設計された TUI を開きます。PR、Issue、gist のタブを切り替えて、ターミナルから状況を確認します。Type /experimental in the CLI conversation to open the redesigned TUI. Switch among PR, issue, and gist tabs to inspect work from the terminal.

  3. 音声Voice

    音声モードで短い依頼を話すSpeak a short request in voice mode

    音声モードを有効にし、短い依頼を話して入力します。オンデバイス音声認識により、音声そのものは端末上に留まります。Enable voice mode and dictate a short request. On-device speech recognition keeps the audio itself on your machine.

    • 静かな場所で、まずは 1 文の依頼から試します。Start with a one-sentence request in a quiet place.
  4. 定期Recurring

    /every で定期プロンプトを作るCreate a recurring prompt with /every

    /every を使って、毎朝の状況確認や定期的なリマインダーのような小さなプロンプトをスケジュールします。Use /every to schedule a small prompt such as a morning status check or recurring reminder.

  5. 背景Background

    バックグラウンドタスクを走らせるRun a background task

    CLI から時間のかかる確認や調査をバックグラウンドで開始し、ターミナルに戻って別作業を続けます。完了したら結果を確認します。Start a longer check or investigation in the background from the CLI, return to the terminal for other work, then review the result when it finishes.

チェックリストChecklist

  • CLI がターミナルで起動し、同じ GitHub アカウントで認証されている。The CLI starts in the terminal and is authenticated with the same GitHub account.
  • /experimental で PR・Issue・gist のタブ付きビューを開ける。/experimental opens a tabbed view for PRs, issues, and gists.
  • 音声モードで入力した依頼がテキストとして扱われ、音声は端末外に出ない前提で試せる。A request spoken in voice mode is handled as text, with audio kept on your machine.
  • /every の定期プロンプトまたはバックグラウンドタスクを作成し、結果を確認できる。You can create a recurring prompt or background task with /every and inspect its result.
ヒント・応用Tips & variations

もう一歩使いこなすGo a little further

試してみるThings to try

  • ガイド 05 の Automations と組み合わせ、CLI で作った定期プロンプトを日々の確認フローに入れる。Combine this with guide 05, Automations, and fold CLI recurring prompts into your daily check-in flow.
  • PR や Issue の確認は、ブラウザを開く前の一次確認として TUI で済ませる。Use the TUI as a first-pass check of PRs and issues before opening the browser.
  • 音声モードは短い指示の下書きに向いています。長い仕様はテキストで補足します。Voice mode works well for drafting short instructions; add longer specifications in text.
Copilot CLI はアプリと同じ基盤上で動くため、ターミナル中心の作業でもアプリ側の体験とつながります。Because the Copilot CLI runs on the same foundation as the app, terminal-first work can stay connected to the app experience.
