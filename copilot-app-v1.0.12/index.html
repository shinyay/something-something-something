Added 追加

  • エージェントピッカー — カスタムエージェントをセッション開始前・実行中に選択。Issue や PR など各起点から生成されたセッションにも適用されます。Agent picker — choose a custom agent before a session starts or while it runs. It also applies to sessions created from any entry point, such as issues and PRs.
  • 統合ブラウザーでの HTML 表示 — エディターヘッダー、ファイルツリーのコンテキストメニュー、チャットのインライン参照から表示。Vite / React などビルドを要するアプリはローカルファイルから描画できないため「dev server を起動」の案内を表示。agents.discover 未実装の CLI でも検出が正常にデグレードします。HTML in the integrated browser — open it from the editor header, the file-tree context menu, or inline references in chat. Apps that need a build step (Vite, React, etc.) can't render from a local file, so a “start the dev server” hint is shown. Detection degrades gracefully even on CLIs that don't implement agents.discover.
  • チャット画像のコピー — 画像の右クリックで「画像をコピー」。サムネイルとビューアーの両方に対応。Copy chat images — right-click an image and choose “Copy image.” Works in both the thumbnail and the viewer.

Changed 変更

  • ドラフト PR のフルパネル化 — Agent Merge と「Ready for review」に同じ場所からアクセス。Draft PRs get the full panel — reach Agent Merge and “Ready for review” from the same place.
  • 返信の引用（Quote in reply） — 既存の下書き末尾に追記し、インラインコードや書式を保持。Cmd+Shift+' に対応し、引用の次行にカーソルを配置。Quote in reply — appends to the end of an existing draft, preserving inline code and formatting. Bound to Cmd+Shift+', with the cursor placed on the line after the quote.
  • テーマピッカーのアクセシビリティ — 矢印キーで移動、Enter / Space で適用、Escape で既定に戻す。選択したテーマ名を支援技術に読み上げ。Accessible theme picker — move with the arrow keys, apply with Enter / Space, and reset to default with Escape. The selected theme name is announced to assistive technology.
  • Issue テンプレート追従 — 見出し・セクション・HTML コメントを保持したまま生成。Follows issue templates — generates issues while preserving headings, sections, and HTML comments.

Fixed 修正

  • デフォルトブランチのリモート改名による、数百ファイルの幻の差分を修正。Fixed a phantom diff of hundreds of files caused by renaming the default branch on the remote.
  • Windows で Alt+Tab 後にチャット入力欄がフォーカスを失う問題を修正。Fixed the chat input losing focus after Alt+Tab on Windows.
  • Windows でリサイズハンドルがスクロールバーに重なりクリックできない問題を修正。Fixed a resize handle overlapping the scrollbar and blocking clicks on Windows.
  • 未コミットの設定ファイルでは「Open on GitHub」を隠し、「Reveal in file manager」を表示。For uncommitted config files, hide “Open on GitHub” and show “Reveal in file manager” instead.
  • Settings のセクション移動時に、見出しへ確実にフォーカスを移動。Reliably move focus to the heading when navigating between Settings sections.
  • Windows のトレイがメインアイコンを使用し、明暗どちらのタスクバーでもコントラストを確保。The Windows tray now uses the main icon, keeping good contrast on both light and dark taskbars.
  • Markdown をエディターで開いた際の無害な自動書き換え（アンダースコアのエスケープ等）を防止。Prevent harmless auto-rewrites (such as escaping underscores) when opening Markdown in the editor.
Overview

GitHub Copilot App とはWhat is the GitHub Copilot App?

アイデアからプルリクエストまで、GitHub リポジトリ全体のソフトウェア作業を「見つけ・実行し・操縦し・着地させる」ためのエージェントネイティブなデスクトップ体験です。An agent-native desktop experience for finding, running, steering, and landing software work across your entire GitHub repository — from idea to pull request.

ターミナル・エディター・ブラウザータブ・GitHub のページを絶えず行き来することなく、着手すべき作業の発見、エージェントの起動と操縦、進捗のレビュー、変更の着地までを単一のコントロールセンターで完結できます。VS Code の拡張機能ではなく、複数のエージェントを並列に扱うために設計された独立したデスクトップアプリです。各セッションは分離された Git worktree で動くため、バグ調査・Issue 実装・レビュー対応を互いに衝突させずに同時進行できます。Without constantly switching between terminals, editors, browser tabs, and GitHub pages, you can discover the work to pick up, launch and steer agents, review progress, and land changes — all from a single control center. It is not a VS Code extension; it is a standalone desktop app designed to run multiple agents in parallel. Because each session runs in an isolated Git worktree, you can investigate bugs, implement issues, and address reviews at the same time without them colliding.

All Features

全機能ガイドAll features

Copilot App と、その土台となる一つのエージェントランタイムが提供する機能を、領域別に詳しく解説します。ターミナル・クラウド・自分のツールまで、同じ基盤の上で動作します。A detailed, area-by-area guide to what the Copilot App and the single agent runtime beneath it provide. From the terminal to the cloud to your own tools, everything runs on the same foundation.

My Work（マイワーク）My Work

接続したすべてのリポジトリを横断し、進行中のセッション・Issue・プルリクエスト・バックグラウンド自動化を一つのビューで俯瞰。次に注力すべき作業を判断できる、アプリの中心となるダッシュボードです。A single view across every connected repository showing in-progress sessions, issues, pull requests, and background automations. The app's central dashboard for deciding what to work on next.

並列エージェント開発Parallel agent development

各ローカルセッションは、ブランチの実コピーである独立した Git worktree で実行されます。worktree の作成・切り替え・後始末はアプリが自動処理。ブランチやファイルが衝突することなく、複数のエージェントを同時に走らせられます。Each local session runs in its own Git worktree — a real copy of the branch. The app creates, switches, and cleans up worktrees automatically, so you can run several agents at once without branch or file conflicts.

クラウドセッションCloud sessions

GitHub がホストする分離環境でエージェントが作業を継続。どのデバイスからでもセッションを引き継ぎ、リモートで操作できます。ローカルのリソース制約に縛られません。Agents keep working in a GitHub-hosted, isolated environment. Pick up a session from any device and drive it remotely, free of local resource limits.

Canvas（キャンバス）Canvas

人とエージェントの双方向作業サーフェス。プラン・PR・ブラウザー・ターミナル・デプロイ・ダッシュボード・ワークフロー状態を表示します。エージェントが作業中に更新し、開発者はその場で編集・並べ替え・承認・軌道修正が可能。AX（Agent Experience）の第一歩です。A two-way work surface for people and agents. It shows plans, PRs, browsers, terminals, deployments, dashboards, and workflow state. Agents update it as they work, and developers can edit, reorder, approve, and course-correct right there — a first step toward AX (Agent Experience).

繰り返しのプロンプトやワークフローを、スケジュール実行やバックグラウンドタスクへ変換します。v1.0.11 からはカスタム CRON 式に対応。Turn repetitive prompts and workflows into scheduled runs and background tasks. Custom CRON expressions are supported since v1.0.11.

  • CI をグリーンに戻すGet CI back to green
  • レビューコメントへの対応Respond to review comments
  • 条件を満たしたら自動マージAuto-merge once conditions are met

クラウド / ローカル サンドボックスCloud / local sandboxes

エージェントが本番環境に触れずにコードを実行・検証できる、境界のある実行場所を提供します。Bounded places where agents can run and verify code without touching production.

  • ローカル: ファイルシステム・ネットワーク・システム機能を制限した分離環境。ポリシーを集中管理・強制。Local: an isolated environment with restricted filesystem, network, and system access, with policies managed and enforced centrally.
  • クラウド: GitHub がホストする完全分離の ephemeral な Linux 環境。組織がポリシーを定義。Cloud: a fully isolated, ephemeral Linux environment hosted by GitHub, with policies defined by your organization.

クラウドエージェントCloud agent

スケジュール実行、GitHub イベントへの応答、Issue の起票、コメントまで対応。既定では各書き込み操作の前に許可を求め、信頼が築けたら autopilot へ切り替えられます。Runs on a schedule, responds to GitHub events, files issues, and comments. By default it asks permission before each write action; once you trust it, switch to autopilot.

Session Modes

セッションモードSession modes

プロンプト作成欄の枠線・フォーカスリング・送信ボタンの色は、現在のモードに応じて変化します（v1.0.11 以降）。作業の任せ方を場面ごとに切り替えられます。The composer's border, focus ring, and send-button color change with the current mode (since v1.0.11), so you can adjust how much you delegate for each situation.

Shell（シェル）Shell

アクセシビリティAccessibility

スクリーンリーダーはセッションやチャットのエージェント活動（返信・ツール呼び出し・CLI コマンド）を読み上げ、Settings > Accessibility でカテゴリ別にトグルできます。チャットの記録は見出しナビゲーションでメッセージ単位に移動でき（各メッセージを「You said:」「Copilot said:」と読み上げ）、テーマピッカーや検索結果の件数も支援技術に伝えられます。Screen readers announce agent activity in sessions and chats (replies, tool calls, CLI commands), and you can toggle it by category under Settings > Accessibility. Chat transcripts support heading navigation to move message by message (each is announced as “You said:” / “Copilot said:”), and the theme picker and search result counts are conveyed to assistive technology as well.
